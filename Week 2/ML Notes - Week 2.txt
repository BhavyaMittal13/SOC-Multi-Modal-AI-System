Linear Regression
What it does: Finds the best-fit line through data points.
Formula: y = w₁x₁ + w₂x₂ + ... + b
How it learns: Minimizes the Mean Squared Error (MSE) between predictions and actual values. The math behind this is "Ordinary Least Squares" — there's a closed-form solution. (For neural networks, we'll need gradient descent.)
Key insight: The coefficients tell you directly how much each feature matters.
* If coef(area) = 60, then adding 1 sqft increases price by ₹60k.
* If coef(age) = -2000, then each year older decreases price by ₹2000.
Assumptions of Linear Regression (I need to understand these better):
1. Linear relationship between X and y
2. Errors are normally distributed
3. No multicollinearity (features shouldn't be highly correlated with each other)
4. Homoscedasticity (equal variance of errors)

Train-Test Split
Why: If you test on training data, the model has "seen" all the answers. It's like giving a student the exam paper beforehand and being surprised they scored 100%.
Rule: Never touch test data until the very end.
Typical ratios: 80/20 or 70/30 (training/test)
Cross-validation: Better than a single split — splits the data K ways and averages the result. More reliable estimate of true performance.

Evaluation Metrics
Metric
	Formula
	Good when...
	MAE
	mean(
	actual - pred
	MSE
	mean((actual - pred)²)
	Large errors are especially bad
	RMSE
	√MSE
	Same units as target, large-error-sensitive
	R²
	1 - SS_res/SS_tot
	You want a 0-1 normalized score
	R² intuition:
* R² = 1.0 → perfect predictions
* R² = 0.0 → model is as good as always predicting the mean
* R² < 0 → model is worse than the mean — something is very wrong
